03 ‚Äî Design Process

Balancing User Needs with Technical Constraints

Research & Ideation (Week 1-2)

Analyzed competitor platforms (Letterboxd, IMDb, Goodreads) to understand what users loved and what frustrated them. Created user personas representing different types of movie enthusiasts.

  • Competitive analysis of 5 platforms
  • Identified key user pain points
  • Defined core features vs. nice-to-haves

Information Architecture (Week 3)

Mapped out user flows and site structure to ensure intuitive navigation. Prioritized features based on technical feasibility and user impact.

Visual Design & Design System (Week 4-5)

Created a cohesive visual language that felt modern, approachable, and movie-focused. Built a component library for development efficiency.

Frontend Development (Week 6-11)

Collaborated closely with backend developers to implement the UI using Handlebars, Bootstrap, and custom CSS. Ensured responsive design across devices.

  • Built reusable component library
  • Implemented responsive grid system
  • Integrated real-time chat with Socket.IO

Testing & Refinement (Week 12)

Conducted user testing with classmates and friends, gathering feedback on usability and visual appeal. Made final adjustments before launch.

04 ‚Äî Key Design Decisions

Color Palette & Visual Identity

I chose a dark, cinematic color scheme inspired by movie theaters ‚Äî creating an immersive experience that puts the focus on film imagery and user-generated content.

Card-Based Layout

Movie posters naturally fit into cards, creating visual consistency and making content scannable at a glance.

Generous Whitespace

Despite being dark-themed, I used spacing strategically to prevent visual clutter and guide user attention.

Accessible Typography

Selected font pairings with strong contrast ratios and generous sizing for readability on all devices.
